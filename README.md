# レコード管理アプリ

大磯アーカイヴ向けの無料MVPです。iPhoneで写真3枚と最小情報を入力し、48枚マスターと同じ22列へ下書き化して、確認後にCSV/JSONで出力できます。

## 起動

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

ブラウザで `http://127.0.0.1:4173/` を開きます。

## 現在の範囲

- 写真3枚の入力
- 22列マスター形式の下書き作成
- OpenAI画像解析による一次判定下書き作成
- 要確認フラグ
- 山田確認後のマスター承認
- CSV/JSON出力
- POPUP、専門店、保留候補の簡易抽出
- Googleスプレッドシート同期

画像はブラウザのIndexedDB、レコード本文はlocalStorageへ保存します。無料MVPとしてのローカル保存なので、本番運用ではSupabaseやGoogle Drive連携へ移す想定です。

## OpenAI一次判定

写真と補足情報を入れて `AI一次判定して下書き作成` を押すと、盤の候補、価格帯、販売区分、確信度、要確認理由を作り、48枚マスター22列へ変換します。手入力済みのArtist・Title・型番・国はAIで上書きせず、違いがあれば要確認にします。

Discogs Medianは未接続なので `未取得` 固定です。DU、Face、BBQは査定金額ではなく、取扱文脈との相性を参考表示します。盤を一意に特定できない場合、価格は空欄のまま `要照合` になります。

APIキーはブラウザへ保存せず、Vercel FunctionsからOpenAIへ送信します。

### Vercel環境変数

1. Vercelの対象プロジェクトで `Settings` -> `Environment Variables` を開きます。
2. `OPENAI_API_KEY` にOpenAI APIキーを設定します。
3. `APP_ACCESS_TOKEN` にスタッフだけが知る任意のアクセスコードを設定します。
4. 必要な場合だけ `OPENAI_MODEL` を設定します。未設定時は `gpt-4.1-mini` です。
5. Productionを再デプロイします。

AI判定またはSheets同期を最初に押した時だけアクセスコードを入力します。コードはそのタブ内だけに保持されます。

以前の版でブラウザ保存したAPIキーは、新しいキーへローテーションしてください。ローカルの簡易HTTPサーバーでは画面確認だけでき、AI判定はVercel公開URLで実行します。

## Googleスプレッドシート同期

Apps Script側にHTMLファイルは不要です。`Code.gs` だけを使い、WebアプリURLはデータ受信用のAPIとして扱います。

1. Googleスプレッドシートを作成します。
2. `拡張機能` -> `Apps Script` を開きます。
3. `google-apps-script/Code.gs` の内容を貼り付けます。
4. `デプロイ` -> `新しいデプロイ` -> 種類 `ウェブアプリ` を選びます。
5. 実行ユーザーは自分、アクセス権はまず `全員` または `リンクを知っている全員` にします。
6. 発行された `/exec` で終わるURLを確認します。
7. `Sheets同期` を押すと、承認済みで未同期のレコードが40件ずつ `48枚マスター` シートへ追記/更新されます。OpenAI解析済みのレコードは、追加列に `次回確認項目` と `AI解析JSON` も保存されます。

Vercel FunctionsがApps Scriptの応答件数を確認し、受理されたレコードだけを同期済みにします。

## 新しいスプレッドシートへ移行

1. 新しいGoogleスプレッドシートを作成します。
2. 新しいシート側で `拡張機能` -> `Apps Script` を開きます。
3. `google-apps-script/Code.gs` を丸ごと貼り付けます。
4. Webアプリとして新しくデプロイし、`/exec` URLを取得します。
5. Vercelの環境変数 `SHEETS_WEB_APP_URL` に新しい `/exec` URLを設定します。
6. Vercelを再デプロイします。
7. 既存データも新しいシートへ送りたい場合は、`同期状態リセット` を押します。
8. `未同期を送信` を押します。

## 同期テスト

`運用` タブの `テストデータ作成` を押すと、承認済み・未同期のサンプルレコードを1件作成します。そのまま `未同期を送信` を押して、`48枚マスター` に行が追加されるか確認できます。確認後、シート側のテスト行は削除して大丈夫です。

シート側が無反応の時は、`google-apps-script/Code.gs` を貼り直してから、Apps Scriptの `デプロイを管理` -> 対象デプロイの編集 -> `バージョン` で `新バージョン` を選び、再デプロイしてください。保存だけだとWebアプリURLが古いコードのまま動くことがあります。

## 共有URL

公開URLをスタッフへ渡すだけで使えます。Apps Scriptの `/exec` URLはアプリ側に固定しているため、共有先でURLを入力する必要はありません。
